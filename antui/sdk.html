Button 按钮

preview 效果预览


usage 使用说明

  • AntUI默认已集成
  • 模块名：widget/button
  • 可选依赖：util/flexbox（布局，amui包已集成）

使用方式

default 基础样式

  • 基础特征
    • 颜色蓝色(blue)
    • 尺寸高度(42px)，宽度100%
    • 标签建议button或者a
蓝色按钮 
<button type="button" class="am-button">蓝色按钮</button>
<a href="#" class="am-button">蓝色按钮</a>

color 颜色扩展

  • 可选颜色(4种)：
    • 蓝色(mode值：blue)
    • 红色(mode值：red)
    • 白色(mode值：white)
    • 黑色(mode值：black)
<button type="button" class="am-button" am-mode="blue">蓝色按钮</button>
<button type="button" class="am-button" am-mode="red">红色按钮</button>
<button type="button" class="am-button" am-mode="white">白色按钮</button>
<button type="button" class="am-button" am-mode="black">黑色按钮</button>

size 尺寸扩展

  • 可选尺寸(5种)：
    • 26px(mode值：tiny)
    • 31px(mode值：little)
    • 36px(mode值：small)
    • 40px(mode值：middle)
    • 42px(default)
<button type="button" class="am-button" am-mode="red tiny">红色26px按钮</button>
<button type="button" class="am-button" am-mode="white little">白色31px按钮</button>
<button type="button" class="am-button" am-mode="black small">黑色36px按钮</button>
<button type="button" class="am-button" am-mode="blue middle">蓝色40px按钮</button>
<button type="button" class="am-button" am-mode="blue">默认42px按钮</button>

disabled 按钮可用性

  • 可选类型(3种)：
    • disabled(通用不可用，最常用)
    • blue-disabled(不推荐，少数场景使用)
    • red-disabled(不推荐，少数场景使用)
<button type="button" disabled="disabled" class="am-button" am-mode="red">正常按钮 不可用</button>
<button type="button" class="am-button" am-mode="red disabled">看起来 不可用</button>
<button type="button" disabled="disabled" class="am-button" am-mode="blue blue-disabled">特殊不可用蓝色按钮</button>
<button type="button" disabled="disabled" class="am-button" am-mode="red red-disabled">特殊不可用红色按钮 </button>

flat 通栏按钮

<button type="button" class="am-button" am-mode="flat warn">警示按钮</button>
<button type="button" disabled="disabled" class="am-button" am-mode="flat warn">警示按钮</button>

light 简易按钮风格

<button type="button" class="am-button" am-mode="blue light small">蓝色简易按钮</button>
<button type="button" class="am-button" am-mode="white light small">白色简易按钮</button>
<button type="button" class="am-button" am-mode="light small disabled">不可用简易按钮</button>

layout 布局

  • 可选依赖：util/flexbox（布局，amui包已集成）
行内按钮 最小号按钮 
<div class="am-flexbox" am-mode="average">

    </div>

Modular 属性选择器和模块化

issue 论点阐述——模块化的契机

  • 属性选择器为模块化带来了契机
    • 解决方案： 一个很小的技术尝试，引入两个目前不太敢使用的选择器，:not()伪类，[xxx]属性选择器。
      • 关于版本控制，比如am-tab组件，他有8.2，8.3，8.4，taobao版，那么仅仅<div class=”am-tab” am-version=”8.2”>，通过am-version控制。
      • 关于不同版本组件并存，同时使用<div class=”am-tab” am-version=”8.2”>，<div class=”am-tab” am-version=”taobao”>，背后的css实现原理基于[xxx]属性选择器，为.am-tab[am-version=”8.2”]，.am-tab[am-version=”taobao”]，各自在各自名称空间（am-version）下写css，无冲突
      • 覆盖即更新，最简单的，我肯定不想带版本，直接这样使用，<div class=”am-tab”>，我不带名称空间写css，.am-tab{}，这好像有点不对，这个样式引入，就会污染.am-tab[am-version=”8.2”]带名称空间的。幸运的是，有:not()伪类可以解决该问题，.am-tab:not([data-version])，标注没有名称空间的情况。
    • 这以前我们使用不多，万一出问题了呢，做技术，必须大胆尝试，小心验证。
      • 看看caniuse上这两个属性兼容性怎么样——哇擦，Android2.1就支持，而且一路标绿，这可比某些橙色部分支持厉害多了。
      • 当然，只看canisue也不行，不放心，我肉身进入我们设备实验室，测了20多款机器，Android2.2，Android2.3，Android4.1，YunOS、iOS、Flyme OS等等系统，小米（红、2）、锤子、三星、摩托罗拉、中兴、vivo、卓普、魅族、HTC等等。等等，一看怎么都还是好的。顺便验证了gradient的三种写法的支持，最老写法-webkit-gradient基本都支持，中间写法-webkit-liner-gradient有些不支持，最新写法liner-gradient基本都不支持。
    • 结果太美好，我们该不该下狠手试试？

why 关于为何引入新方案

  • 方案孵化于AMUI 8.4版，作为探索型新方案，致力于版本并存、覆盖即更新、组件库丰富

  • 目前现状：

    • AMUI的设计风格跟随钱包native设计风格，一个版本改一次，变更频繁，且部分组件，可能需要涉及结构重新改造，更新成本大。
    • 实际使用中，对应业务线都对AMUI的更新换代持谨慎态度，现有已上线产品，本来以稳定上线，AMUI换代，难免产生会引入风险因素的心理。
    • AMUI版本过多，8.0版、8.1版、8.2版、8.3版，大版下面还可能有小版，以实际项目举例，上次天猫包项目中，甚至有使用8.0版AMUI的
    • AMUI自身发展有限，组件有限几个，且被固化在频繁的native风格改版，组件无积累。
    • 同名组件，很容易彼此冲突
    • ……
  • AMUI想做出一些改变，引入组件版本机制，解决版本控制、不同版本组件并存、覆盖即更新。
    • 做到版本控制，小可称为版本控制，大即为风格衍生，同一组件，不同长相，组件库会更丰富，而每一个钱包改版，只是为我们积累更多优秀组件
    • 做到不同版本组件并存，同类组件，如am-tab，但是由8.4版，tabao.xx版，通过版本，作为其同名（am-tab）的额外名称空间，防止冲突。不仅css可以不受限制，html结构也能做到自由灵活。
    • 覆盖即更新，大部分同学使用am-tab，都是希望我引用8.2版的时候，是8.2版，引入8.3版的时候，自然变成8.3版，这我就能特别放心使用amui了。

other 其它

  • 各类升级预案：

    • Old is better：主文件保持老的，部分新样式，指定使用新版本
    • New is better：主文件保持新的，部分老样式，指定使用老版本
    • More is better：无主文件，从丰富的组件库中拼凑出自己的版本。

  • 最后，总结一个，做移动前端真苦逼。不同手机机型，不同系统，都有各种坑。甚至，同个手机上，UC浏览器，原生浏览器等，就还真能表现的不一样。对很多技术，都只能保持保守态度。
    但，我们该大胆尝试和探索。。当然，最好也要多验证，做后备保险，小心使得万年船。。。不然死的还是你。。

usage 使用方法

  • AntUI默认已集成
  • 模块名：base/reset

使用方式

issue 论点中立

虽然文件名为reset，其实只是精简版的reset。

reset和normalize的区别可以看出，一个是控制倾向，一个是自然倾向。

精简版reset中，主要保留的是一些布局类reset（如margin:0），去掉了控制倾向比较强的部分。

同时，一些不常用的html5标签，也被去除。

  • 改造的目标
    • 最低控制 用户没有重置类样式时，也能保证基本显示
    • 自由兼容 使用者可以自由额外引入reset or normalize，而不用担心冲突
    • 节约代码 去掉一些不必要的代码，节约是良好的美德
